# Step 4: Report Assembly

## Objective
Assemble the previous step's output — which contains both the narrative executive summary and the analysis data tables — into a single, well-structured markdown report. This is the final deliverable of the pipeline.

## Guidelines
- Structure the report in the following order:
  1. Title page block: report title, period, and preparation date
  2. Table of contents listing the sections below
  3. Executive Summary: the prose narrative, reproduced exactly with no edits
  4. Financial Analysis: the financial summary and revenue and expenses tables from the Analysis Data section
  5. Customer Analysis: the customer metrics, top/bottom growth areas, and flagged items from the Analysis Data section
- Do not add, remove, or alter any figures, sentences, or table values from the input. Assembly only — no new content.
- Use consistent markdown heading levels: `#` for the report title, `##` for top-level sections, `###` for subsections.
- Add a horizontal rule (`---`) between each major section for readability.
- The table of contents should use markdown links to the section headings.
